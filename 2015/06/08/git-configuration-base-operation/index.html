Git配置

Git的配置可以分三个级别System、Global、Local。

System为所有用户的配置，对于文件/etc/gitconfig。

Global为对应当前用户配置，对应文件~/.gitconfig。

Local只针对当前的项目，对应于.git/config，只对当前项目起作用。

提交已暂存的文件

使用git commit提交文件，这种方式会打开文本编辑器来输入提交信息。

可以使用git commit -m "message"这种方式来提交。

移除文件

要从Git中移除某个文件，就必须要从已跟踪文件清单中移除（确切地说，是从暂存区移除），然后提交。

首先使用git rm删除文件再使用git commit文件才真正不纳入版本管理。

如果文件已经加入到暂存区，则应该加-f选项，如git rm -f [file]。

如果只是将文件从暂存区删除，命令为：git rm --cached [file]。

移动文件

不像其他的版本控制系统，Git并不跟踪文件移动操作。如果在Git中重命名了某个文件，仓库中存储的元数据并不会体现出这是一次改名操作。

.gitignore文件

.gitignore文件可以忽略掉不想加入版本管理的文件，如IDE的配置文件或编译生成的缓存文件这些。格式规范如下：

  • 所有空行或者以注释符号 ＃开头的行都会被Git忽略
  • 可以使用标准的glob模式匹
  • 匹配模式最后跟反斜杠（/）说明要忽略的是目录
  • 要忽略指定模式以外的文件或目录，可以在模式前加上惊叹号（!）取反
