The power of habit

发表于 2015-06-15   |   分类于 Grow up   |  

我们为什么这样生活，那样工作

经常会有这样的一种感觉——天啊，我的一天是怎么过的？

而事实上，一天24小时不变，一天基本的生活作息也都差不多。产生这种反差的原因是我们做的很多事情都是“不经过大脑”的。

所有人的生活都有其明确的形态，但其实都是由各种各样的习惯构成的。

习惯之所以出现是因为大脑一直在寻找一种省力的方式，这样我们几乎可以同时处理更多的信息。

习惯对我们生活的影响程度超过我们的认识，让大脑依赖它们，同时将逻辑等其他一切排除在外（无意识）。

习惯回路——暗示、惯常行为和奖赏

  • 存在着一个暗示，能让大脑进入某种自动行为模式，并决定使用哪种习惯
  • 存在一个惯常行为，可以是身体的、思维或情感方面的
  • 奖赏，让你的大脑辨别出是否应该记下这个回路

习惯回路中的暗示几乎可以是所有的事物，视觉、特定的时间地方、某种情绪、一系列的思绪或者特定人的陪伴；习惯行为可以让人觉得不可思议的复杂，也能令人觉得异常简单；奖赏则涵盖了让人感官得到满足的事物，比如酒精。当然奖赏也可以是那种让人沾沾自喜的感觉。

习惯的力量很强大，但习惯本身却很脆弱

渴求——创造新习惯

正是欲望，驱动着习惯回路

  1. 找出一种简单又明显的暗示
  2. 清楚地说明有哪些奖赏

有些东西，在长期接触下，早就让你习以为常，你根本不知道它的存在。所以要改变一个习惯，或者创造一个习惯，也就没有了明确的暗示，甚至所谓的奖赏也并没有被意识到。

习惯的力量来源于其创造出的神经渴求。在大部分时间里，这些渴求是逐渐产生的，而我们确实没有意识到它们的存在，所以往往看不到它们的影响。

习惯是这样产生的：把暗示、惯常行为和奖赏拼在一起，然后培养一种渴求来驱动这一回路。

习惯的改变

要改变习惯，你必须留住旧习惯回路中的暗示，提供旧习惯回路中的奖赏，但要插入一个新的惯常行为。

喝醉本身并不属于渴求。酗酒者之所以想喝酒，是因为酒让他们能逃避现实，放松心情，又有人陪伴，而且缓解焦虑，情感也能得到释放。

  1. 第一步，找到习惯引发的暗示
  2. 第二步，分析惯常行为带来的奖赏
  3. 第三步，用一个新的行为去代替要改变的惯常行为，它有着同样的暗示和奖赏

在有意寻找驱动我们行为的渴求感之前，我们通常并真正理解它们。

事实上，大脑是可以接受重新编排的，你需要做的仅仅是刻意为之。

在突然经历压力过大的情况时，新养成的习惯可能又会被打回了原形，而其中的原因是缺少信仰。你可能都不相信自己可以改变。

一旦人们学会信仰某种东西，这种信仰就会扩展到生活的其他方面，直到他们开始相信自己能改变。信仰是将改造过的习惯回路变成永久性行为的要素。

你要相信的不一定是上帝，但你必须相信事情会变好转。

实践

  1. 找到惯常行为（你要改变的东西），暗示是什么？奖赏又是什么？
  2. 用奖赏做实验（你渴望得到什么），改变你的惯常行为之后，感受如何？
  3. 分隔出暗示（你什么想这么做），诱因可以分为几类：
    1. 地点
    2. 时间
    3. 情绪状态
    4. 其他人
    5. 之前紧挨着的动作
  4. 制订计划，选择能够满足你所渴求的奖赏行为，你就可以改善你的习惯，并重复做下去。

文件的状态

对于Git而言，文件有三种状态：

  • 已修改（Modified）
  • 已提交（Committed）
  • 已暂存（Staged）

新建一个learning git.txt文件，使用git status查看时，Git会提示有未被追踪（Untracked）的文件。

当使用git add添加时，文件的状态就会变为已暂存（也称已索引）。

当调用git commit时，才会把已暂存的文件提交到git仓库中。

也就是说，在工作目录（Working Directory）下的文件就两种状态：已追踪和未追踪。而已追踪的文件又可分为已修改和未修改两种情况。工作目录下的文件生命同期如图：

关于版本控制系统

版本控制系统（Version Control System）记录一个或一系列文件随时间的推移变化。如果需要，可以随时将文件还原到之前的某一状态。并且，版本控制系统记录了文件的所有变更记录，可以方便地找出谁操作了文件、谁引入了问题（issue）等等。

集中式版本控制系统（Centralized Version Control Systems）

本地版本控制系统的最大缺点就是无法与他人协同工作，因此集中式版本控制系统应运而生。 其中有著名的CVS、Subverion等。集中式版本控制系统的工作原理如下：

可以看到整个系统分为了服务器和客户端两个部分。服务器维护了一份最新、最完整的文件。客户端的三个最基本操作：取（Checkout）、更（Update）、提（Commit）。

分布式版本控制系统（Distributed Version Control Systems）

想像一下，对于集中式版本控制系统而言，如果很不幸，服务器挂了，带来的结果可能是毁灭性的。再者，假如服务器当机了几个小时（这还挺常见的），这段时间内是无法对服务器进行任何操作的。

分布式版本控制系统也有服务器与客户端之分，所不同的是，每一个客户端都是仓库（Repository）的镜像。同时，每一个客户端都具有独立完整的版本控制能力。就算服务器挂了，任意一台客户端都有能力将所有文件还原回去。结构如下：

Git的设计目标

Git最初由Linux社区开发，用来维护Linux内核代码的。其一开始设计就考虑了以下问题：

  • Speed 高速
  • Simple design 设计简单
  • Strong support for non-linear development(thousands of parallel branches) 强力支持非线性开发，支持成千上万的平行分支
  • Fully distributed 完全分布式
  • Able to handle large projects like the Linux kernal efficiently(speed and data size) 能够处理像Linux内核这样的大量项目，要求高速和大数据量。

Git的特点

记录文件快照而非文件差异：

其他版本控制系统，像Subversion，每次记录的是实际上是新版本文件相较于上一版本的差异。其工作原理图如下：

而Git的原理并非如此，对于每一次的提交，Git实际上是对所有文件进行了一次快照。每一次的快照保存的是文件版本的引用。如果文件没有改变就指向原来的文件，否则指向新文件。工作原理如下：

几乎所有操作都是本地执行的：

由于每一个客户端都拥有完整的服务器文件镜像，所以绝大多数的操作只需要在本地就能执行。查询日志、对比文件差异这种操作都只要在本地就能执行。甚至在没有网络的情况下也可以先将文件提交到本地系统，等连上服务器再进行推送。

保持数据的完整性：

文件在保存到Git之前都要时行校验和（Checksum）计算，该结果将作为数据的唯一标识和索引。

Git使用的校验和算法为SHA-1，通过对文件的内容或目录结构计算出一个40位的十六进制字符串。Git数据库就是使用该值作索引的，而非文件名。

Git管理文件的三种状态：

在Git内部，一个文件可能有三种状态：已提交（Commited）、已修改（Modified）、已暂存（Staged）。

快捷键

一般的快捷键可以分为以下几类：

  • 编辑（Edit）
  • 选择（Selection）
  • 跳转（Go to）
  • 查找（Find）
  • 标签（Tabs）
  • 视图（View）
  • 工程（Project）
  • 通用（General）

编辑（Edit）

  • CMD + [、] 改变缩进
  • CMD + CTRL + 上、下 交换行位置
  • CMD + SHIFT + D 复制与粘贴
  • CMD + 左、右 行开头与结尾
  • CMD + / 注释
  • CMD + CTRL + / 块注释
  • CTRL + K 从光标删除到行尾
  • CTRL + SHIFT + K 删除行，CMD + X有类似效果
  • CMD + OPTION + T 插入特殊符号

光标选中（Selection）

  • CMD + D 选中相同的词
  • CMD + CTRL + G 多重文本选中
  • OPTION + 鼠标选择或鼠标中键选择 块选择
  • CTRL + SHIFT + M 选中当前匹配括号中的内容

查找（Find）

  • CMD + F 查找
  • CMD + OPTION + F 替换
  • CMD + G 查找下一处
  • CMD + SHIFT + F 从文件在查找
  • CMD + E 缓存用于查找的内容
  • CMD + SHIFT + E 缓存用于替换的内容
  • CMD + OPTION + E 替换下一个

跳转（Go to）

  • CMD + P 文件跳转
  • CMD + R 定位文件中的方法
  • CMD + G 定位行
  • CTRL + M 定位匹配括号
  • F2 下一个书签
  • CMD + F2 开关书签
  • CMD + SHIFT + F2 清除所有书签
  • CMD + OPTION + 左、右 打开文件跳转

通用

  • CMD + SHIFT + P 命令
  • CMD + K,CMD + B 开关侧边栏

标签（Tabs）

  • CMD + SHIFT + T 最后关闭的tab
  • CTRL + TAB tab之间切换
  • CMD + 数字　切到相应的tab
